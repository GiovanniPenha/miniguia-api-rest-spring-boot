# 📖 Sobre o projeto

Este projeto foi desenvolvido durante um desafio da **DIO** com o objetivo de explorar o **NotebookLM** como ferramenta de aprendizagem ativa.

Para isso, foi criado um caderno temático baseado em documentação oficial sobre **APIs REST** e **Spring Boot**, utilizando Inteligência Artificial para organizar informações, responder perguntas e consolidar o aprendizado.

Mais do que concluir um desafio, a proposta foi construir um material de consulta que poderá ser utilizado futuramente durante o desenvolvimento de projetos backend.

---

# 🎯 Objetivos

- Compreender os fundamentos de APIs REST.
- Entender como funciona a comunicação através do protocolo HTTP.

- Relacionar esses conceitos com o desenvolvimento utilizando Spring Boot.

# 📒 Miniguia de Estudo

## O que é uma API?

Uma API (Application Programming Interface) é uma interface que permite a comunicação entre diferentes sistemas por meio de regras bem definidas.

---

## O que é REST?

REST (Representational State Transfer) é um estilo arquitetural utilizado para construir APIs baseadas em recursos, utilizando os métodos do protocolo HTTP.

---

## HTTP

HTTP é o protocolo responsável pela comunicação entre cliente e servidor através de requisições e respostas.

---

## Métodos HTTP

| Método | Finalidade |
|---------|------------|
| GET | Consultar informações |
| POST | Criar novos recursos |
| PUT | Atualizar completamente um recurso |
| PATCH | Atualizar parcialmente um recurso |
| DELETE | Remover um recurso |

---

## Principais códigos de status

### Sucesso

- 200 OK
- 201 Created
- 204 No Content

### Erros do cliente

- 400 Bad Request
- 401 Unauthorized
- 403 Forbidden
- 404 Not Found

### Erros do servidor

- 500 Internal Server Error

---

## Spring Boot

As principais anotações utilizadas para criação de APIs REST são:

- `@RestController`
- `@RequestMapping`
- `@GetMapping`
- `@PostMapping`
- `@PutMapping`
- `@DeleteMapping`

# 📖 Glossário

| Termo | Significado |
|--------|-------------|
| API | Interface que permite comunicação entre sistemas |
| REST | Estilo arquitetural para desenvolvimento de APIs |
| HTTP | Protocolo utilizado na comunicação cliente-servidor |
| Endpoint | Endereço onde um recurso pode ser acessado |
| Resource | Informação disponibilizada por uma API |
| JSON | Formato utilizado para troca de dados |
